# 記事作成から投稿までの流れ

## 全体フロー

```
Step 1: 記事の内容をGPTに伝える
    ↓
Step 2: GPTが記事タイプを判定する
    ↓
Step 3: GPTが記事本文を生成する
    ↓
Step 4: 内容を確認・修正する
    ↓
Step 5: GPTがWordPressへ下書き投稿する
    ↓
Step 6: WordPress管理画面で最終確認して公開する
```

---

## 各ステップの詳細

### Step 1: 記事の内容をGPTに伝える

ChatGPTのカスタムGPTに、書きたい内容をテキストまたは音声で伝えます。

**例（AI活用記事の場合）:**
```
「ClaudeとCoworkを使ってGitHubに記事を自動投稿する仕組みを作った話を記事にして」
```

**例（通常体験記事の場合）:**
```
「昨日行った〇〇ラーメンの体験記を書いて。麺は細麺で、スープは濃厚醤油だった」
```

---

### Step 2: GPTが記事タイプを判定する

GPTは入力内容から以下のどちらかを自動判定します：

**AI活用記事と判定される内容:**
- ChatGPT、Claude、GeminiなどのAIツール
- 自動化、WordPress連携
- プロンプト作成、AI画像生成
- AIを使って何かを試した記録

**通常体験記事と判定される内容:**
- グルメ、ラーメン、飲食店レビュー
- 商品レビュー、買ってみた記録
- お出かけ、日常記録
- 店舗紹介、観光・イベント

---

### Step 3: GPTが記事本文を生成する

判定したタイプに応じて、LivetubeSTARブログのスタイルで記事を生成します。

**AI活用記事の構成:**
1. 冒頭あいさつ（固定文）
2. 今回の記事テーマ
3. 最初に結論
4. 今回やったこと
5. 実際のAIとのやり取り（HTML会話バブル形式）
6. このあとどう進んだか
7. 実際にできた結果
8. やってみて感じたメリット
9. 注意点・気になったこと
10. まとめ
11. 次の目標

**通常体験記事の構成:**
1. 冒頭あいさつ（固定文）
2. 今回のテーマ
3. 最初に感想や結論
4. 今回行ったこと・食べたもの・見たもの
5. 実際に感じたこと
6. 良かったところ
7. 気になったところ
8. まとめ
9. 次にやりたいこと

---

### Step 4: 内容を確認・修正する

GPTが記事案を提示します。気になる箇所があれば修正を依頼できます。

```
「3段落目をもう少し詳しく書いて」
「タイトルを少し柔らかい表現にして」
「会話ログのパートを省略して」
```

---

### Step 5: GPTがWordPressへ下書き投稿する

確認が完了したら、GPTがWordPress REST APIを通じて下書きとして投稿します。

**送信されるデータ:**
- タイトル
- 本文（HTML形式）
- スラッグ（URL用の英語文字列）
- カテゴリ
- タグ
- ステータス（`draft` = 下書き）

**APIエンドポイント:**
```
POST https://your-wordpress-site.com/wp-json/lts/v1/create-post
X-API-Key: your-api-key
```

---

### Step 6: WordPress管理画面で最終確認して公開する

1. WordPress管理画面 → 投稿 → 下書き
2. 生成された記事を確認
3. 必要に応じてアイキャッチ画像を設定
4. 問題なければ「公開」ボタンを押す

---

## 記事更新フロー

既存の記事を修正する場合：

```
「記事ID 123 のまとめ部分を更新して」
    ↓
GPTが update-post アクションを実行
    ↓
POST /wp-json/lts/v1/update-post
{ "post_id": 123, "content": "..." }
    ↓
WordPress側で記事が更新される
```

---

## カテゴリ・タグの自動処理

- 既存のカテゴリ・タグ名を指定すれば自動で紐付け
- 存在しないカテゴリ・タグは自動で新規作成
- ID指定でも名前指定でも対応

```json
{
  "category_names": ["AI活用", "ChatGPT"],
  "tag_names": ["WordPress", "自動投稿", "GPT"]
}
```
